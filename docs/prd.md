## 一、产品概述

### 1.1 产品愿景

打造一款**桌面端专注监督工具**，通过摄像头实时监控用户注意力状态 + 桌面应用切换检测，结合养成系游戏化机制（公仔成长 + 等级体系），让用户在"被监督"的心理约束下逐步提升自制力，最终将"专注"内化为习惯。

### 1.2 核心价值主张

| 用户痛点 | 我们的解决方案 |
|---------|--------------|
| 工作时忍不住刷社交媒体/视频 | 桌面应用监控，切换分心应用时实时提醒 |
| 缺乏外部约束，自律困难 | 摄像头监控，制造"被看着"的监督感 |
| 专注后没有成就感，难以坚持 | 公仔养成 + 等级成长，专注成果可视化 |
| 自制力提升无法量化 | 自制力等级系统，长期追踪成长曲线 |

### 1.3 目标用户

**核心用户：**
- 18-35 岁知识工作者（程序员、设计师、写作者等）
- 需要长时间面对电脑学习的学生群体
- 有 ADHD 倾向或注意力容易分散的人群

**次要用户：**
- 自由职业者（需要自我管理）
- 远程办公者（缺乏办公室环境约束）

### 2.2 功能详细说明

#### 2.2.1 摄像头注意力监控

**功能描述：** 通过电脑摄像头实时检测用户的注意力状态。

**检测维度：**
1. **人脸检测：** 判断用户是否在电脑前
2. **视线方向：** 判断用户是否在看屏幕（近似检测）
3. **头部姿态：** 判断是否长时间低头/侧头（可能分心）

**状态判定逻辑：**

| 状态 | 判定条件 | 公仔反馈 |
|------|---------|---------|
| 专注 | 人脸在画面中 + 视线朝向屏幕 + 活动窗口为工作应用 | 公仔开心，经验值增长 |
| 分心 | 人脸在画面中 + 视线偏离屏幕 或 活动窗口为分心应用 | 公仔提醒，经验值暂停 |
| 离开 | 人脸不在画面中超过设定时间 | 公仔等待，计时暂停 |

**技术实现要点：**
- 使用本地 AI 模型（如 MediaPipe、ONNX Runtime）
- 所有处理在本地完成，不上传任何图像数据
- 提供"无摄像头模式"，仅使用桌面监控

---

#### 2.2.2 桌面应用监控

**功能描述：** 追踪用户当前使用的应用程序，识别分心行为。

**核心功能：**

1. **活动窗口追踪：** 实时记录用户当前聚焦的应用窗口
2. **应用分类：**
   - 工作应用（白名单）：如 IDE、文档、设计软件、邮件客户端
   - 分心应用（黑名单）：如社交媒体、视频网站、游戏
   - 中性应用：未分类的应用
3. **切换检测：** 记录应用切换频率，高频率切换可能表明注意力分散

**分心判定逻辑：**

```
IF 当前活动窗口 ∈ 分心应用黑名单 THEN
    触发"分心状态"
    记录分心时长
    公仔状态 → 担忧/提醒
    发送桌面通知
END IF
```

**用户配置：**
- 用户可自定义黑名单/白名单
- 提供"严格模式"：分心应用自动最小化或关闭

#### 2.2.3 公仔养成系统

**功能描述：** 通过养成系虚拟公仔，将用户的专注行为具象化为公仔的成长。

**公仔生命周期：**

```
蛋期 → 幼体期 → 成长期 → 成熟期
（需专注时长解锁下一阶段）
```

**公仔状态系统：**

| 状态 | 触发条件 | 视觉表现 | 影响 |
|------|---------|---------|------|
| 开心 | 持续专注 | 表情愉快、动画活泼 | 经验值加成 |
| 饥饿 | 长时间未专注或未互动 | 肚子咕咕叫 | 经验值减少 |
| 生病 | 频繁分心 | 打喷嚏、精神萎靡 | 需要专注"治疗" |
| 沉睡 | 非工作时段 | 闭眼睡觉 | 不影响经验值 |
| 期待 | 用户归来 | 摇尾巴/挥手 | 提示开始工作 |

**公仔互动功能：**
- 抚摸（点击/拖拽）：提升亲密度
- 喂食（消耗金币）：恢复状态
- 装扮：使用金币购买装饰品
- 多公仔切换：解锁后可选择不同造型

**成长阶段解锁条件：**

| 阶段 | 累计专注时长 | 解锁内容 |
|------|------------|---------|
| 蛋期 | 0 小时 | 初始状态 |
| 幼体期 | 10 小时 | 基础互动解锁 |
| 成长期 | 50 小时 | 装扮系统解锁 |
| 成熟期 | 150 小时 | 繁衍系统（可选） |

#### 2.2.4 等级系统

**功能描述：** 将用户的专注行为量化为等级，体现"自制力成长"。

**等级体系：**

| 等级范围 | 称号 | 累计专注时长 |
|---------|------|------------|
| Lv.1-5 | 新手学徒 | 0-25 小时 |
| Lv.6-10 | 专注学徒 | 25-75 小时 |
| Lv.11-20 | 自律达人 | 75-200 小时 |
| Lv.21-30 | 专注大师 | 200-400 小时 |
| Lv.31-50 | 意志宗师 | 400+ 小时 |

**经验值获取规则：**

| 行为 | 经验值 |
|------|-------|
| 专注 1 分钟 | +1 XP |
| 连续专注 30 分钟 | +50 XP（奖励） |
| 完成当日专注目标 | +100 XP |
| 连续打卡 7 天 | +200 XP |
| 分心超过 5 分钟 | -10 XP |

**解锁内容：**
- 新等级解锁：公仔皮肤、装饰品、背景主题
- 称号展示：在应用内显示当前称号

#### 2.2.5 实时干预与提醒

**功能描述：** 当检测到分心行为时，通过多种方式进行温和干预。

**提醒方式：**

| 触发场景 | 提醒方式 | 频率限制 |
|---------|---------|---------|
| 切换到分心应用 | 桌面通知 + 公仔表情变化 | 首次立即提醒，之后每 2 分钟 |
| 视线长时间偏离屏幕 | 公仔发出声音提醒 | 每 1 分钟 |
| 离开座位超过 5 分钟 | 桌面通知"公仔在等你" | 返回时提醒 |
| 专注 30 分钟 | 鼓励通知 + 金币奖励 | 每达到时提醒 |

**提醒文案示例：**
- "你的公仔在担心你，快回来工作吧！"
- "已经专注 30 分钟了，太棒了！获得 10 金币"
- "检测到你打开了社交媒体，还记得今天的专注目标吗？"

**可配置项：**
- 提醒强度：温和/标准/严格
- 提醒声音：多种可选
- 勿扰模式：设定勿扰时段

#### 2.2.6 番茄钟 + 强制放松机制 ⭐ 新增功能

**功能描述：** 引入番茄工作法，工作一段时间后强制进入放松时段，帮助用户保持长期注意力，避免疲劳累积。

**核心机制：**

```
┌─────────────────────────────────────────────────────────────┐
│                        番茄钟周期                             │
│                                                             │
│   工作 25min ──→ 短休息 5min ──→ 工作 25min ──→ 短休息 5min   │
│        │                                    │              │
│        └────────── 4 个番茄后 ───────────────┘              │
│                          ↓                                  │
│                    长休息 15-30min                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**时段配置（用户可自定义）：**

| 时段类型 | 默认时长 | 可调范围 | 说明 |
|---------|---------|---------|------|
| 工作时段 | 25 分钟 | 15-60 分钟 | 专注时段，监控激活 |
| 短休息 | 5 分钟 | 3-15 分钟 | 强制放松，监控暂停 |
| 长休息 | 15 分钟 | 10-30 分钟 | 每 4 个番茄后触发 |

**强制放松机制：**

工作时段结束后，系统会自动进入放松模式，用户**必须**休息：

| 机制 | 说明 |
|------|------|
| 屏幕变暗 | 放松时段屏幕亮度降低 30-50% |
| 公仔提示 | "该休息啦！让我陪你放松一下吧~" |
| 放松内容推荐 | 护眼提示、拉伸动画、呼吸引导 |
| 跳过惩罚 | 强制跳过休息会扣除公仔生命值/经验值 |
| 不可跳过模式 | 严格模式下，休息时段不可跳过 |

**放松时段活动建议：**

| 活动 | 描述 | 公仔互动 |
|------|------|---------|
| 护眼休息 | 距离屏幕 50cm，远眺 20 秒 | 公仔闭眼动画 |
| 拉伸运动 | 颈部/肩部拉伸引导动画 | 公仔做体操 |
| 呼吸练习 | 4-7-8 呼吸法引导 | 公仔一起呼吸 |
| 喂食互动 | 给公仔喂食，恢复状态 | 公仔吃东西动画 |
| 装扮时间 | 为公仔换装 | 公仔开心转圈 |

**用户配置：**

| 配置项 | 选项 |
|-------|------|
| 番茄钟开关 | 开启/关闭 |
| 工作时长 | 15/25/30/45/60 分钟 |
| 短休息时长 | 3/5/10/15 分钟 |
| 长休息时长 | 10/15/20/30 分钟 |
| 强制休息模式 | 温和提醒 / 强制执行 / 不可跳过 |
| 放松内容偏好 | 护眼/拉伸/呼吸/自由活动 |

**心理学依据：**
- 番茄工作法已被证明能有效提升专注力和减少疲劳
- 强制休息防止"过度专注"导致的注意力衰退
- 休息时的公仔互动增强情感连接，形成正向循环

#### 2.2.7 扣血特效与视觉反馈系统 ⭐ 新增功能

**功能描述：** 当检测到用户分心（切换到黑名单应用、视线偏离、离开座位）时，屏幕边缘会出现类似游戏"扣血"的红色特效，提供即时视觉反馈，增强"被监督感"。

**视觉反馈机制：**

```
┌─────────────────────────────────────────────────────────────┐
│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│
│▓                                                           ▓│
│▓                                                           ▓│
│▓                                                           ▓│
│▓                    正常工作区域                             ▓│
│▓                                                           ▓│
│▓                                                           ▓│
│▓                                                           ▓│
│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│
└─────────────────────────────────────────────────────────────┘
        ↑ 分心时边缘出现红色"扣血"特效，动态闪烁
```

**特效触发条件：**

| 触发场景 | 特效表现 | 持续时间 |
|---------|---------|---------|
| 切换到黑名单应用 | 边缘红光闪烁 + 公仔担忧表情 | 应用切换期间持续 |
| 视线长时间偏离屏幕 | 边缘红光渐变 + 公仔呼喊动画 | 视线偏离期间持续 |
| 离开座位 | 边缘淡红色静态光 | 离开期间持续 |
| 公仔生命值低 | 边缘持续微弱红光警示 | 直至状态恢复 |

**特效强度档位（用户可调）：**

| 强度 | 描述 | 适用人群 |
|------|------|---------|
| 关闭 | 无视觉特效，仅公仔反馈 | 不喜欢干扰的用户 |
| 轻微 | 边缘微弱红光，不闪烁 | 敏感型用户 |
| 标准 | 边缘红光 + 缓慢脉冲闪烁 | 大多数用户 |
| 强烈 | 边缘亮红光 + 快速闪烁 + 音效 | 需要强监督的用户 |

**动态变化规则：**

```
分心时长 0-10 秒   → 轻微红光
分心时长 10-30 秒  → 红光增强 + 脉冲频率加快
分心时长 30-60 秒  → 强烈红光 + 快速闪烁 + 声音提醒
分心时长 >60 秒    → 公仔"生病"状态 + 经验值扣除
```

**配色与可访问性：**
- 主色：#FF4444（警告红）
- 辅助色：#FFAA00（提醒橙）
- 支持"色盲友好模式"：红光替换为蓝色/黄色
- 特效透明度：20-60%，不影响正常工作

**技术实现要点：**
- 使用系统级透明窗口叠加层
- 不影响其他应用的正常使用
- GPU 加速，避免性能损耗

#### 2.2.8 行为轨迹日志与时间线分析 ⭐ 新增功能

**功能描述：** 全天候记录用户的应用切换轨迹、窗口标题（需授权），生成详细的时间线报告，帮助用户了解"我这一天到底干了什么"，并为后续改进提供数据支撑。

**数据采集层级（隐私分级）：**

| 级别 | 采集内容 | 隐私等级 | 默认开启 |
|------|---------|---------|---------|
| **L1 基础** | 应用名称 + 使用时长 | 低 | ✅ |
| **L2 标准** | 应用名称 + 窗口标题 + 使用时长 | 中 | ⚠️ 需授权 |
| **L3 详细** | 应用名称 + 窗口标题 + URL（浏览器）+ 使用时长 | 高 | ❌ 需明确授权 |
| **L4 完整** | L3 + 屏幕截图采样（模糊化）| 极高 | ❌ 需明确授权 + 密码确认 |

**时间线可视化示例：**

```
┌─────────────────────────────────────────────────────────────┐
│                      今日时间线 (2025-05-20)                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  09:00 ├─ VS Code ──────────────────────────── 1h 23m ──→   │
│        │  └─ 编写用户认证模块代码                             │
│                                                             │
│  10:25 ├─ 微信 ─────────────────────────────── 18m ──→      │
│        │  └─ 与客户对接项目进度                               │
│                                                             │
│  10:43 ├─ Obsidian ────────────────────────── 32m ──→       │
│        │  └─ 整理产品需求文档                                 │
│                                                             │
│  11:15 ├─ Chrome ──────────────────────────── 45m ──→       │
│        │  ├─ GitHub Issues ───── 15m                        │
│        │  ├─ Stack Overflow ──── 20m                        │
│        │  └─ B站（分心） ────── 10m ⚠️                        │
│                                                             │
│  12:00 ├─ 午休 ────────────────────────────── 1h ──→        │
│                                                             │
│  ...                                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**统计报告维度：**

| 维度 | 数据项 | 用途 |
|------|-------|------|
| **应用维度** | 各应用使用时长、占比、切换次数 | 了解时间分配 |
| **任务维度** | 按窗口标题聚合的任务时长 | 了解具体做了什么 |
| **效率维度** | 专注时长 vs 分心时长 vs 休息时长 | 自制力评估 |
| **时段维度** | 高效时段 vs 低效时段分析 | 优化工作节奏 |

**AI 分析层（可选功能）：**

用户可选择开启 AI 分析，让大模型帮助解读行为数据：

| 分析类型 | 说明 | 所需权限 |
|---------|------|---------|
| 自动分类 | 将窗口标题自动归类（工作/沟通/学习/娱乐） | L2+ |
| 效率洞察 | "你上午效率最高，建议安排重要任务" | L1+ |
| 分心归因 | "你分心最多是因为频繁查看微信" | L2+ |
| 改进建议 | "建议将微信沟通集中在固定时段" | L2+ |
| 周报摘要 | "本周专注 23h，较上周提升 15%" | L1+ |

**AI 分析方案（用户可选）：**

| 方案 | 特点 | 隐私 | 成本 |
|------|------|------|------|
| **本地 LLM** | Ollama 运行本地模型 | 数据不离设备 | 需本地算力 |
| **云端 LLM** | OpenAI/Claude API | 数据需上传 | 用户付费/订阅 |
| **混合模式** | 敏感数据本地，非敏感上云 | 平衡 | - |
| **关闭 AI** | 仅基础统计，无智能分析 | 最安全 | 免费 |

**隐私保护措施：**

1. **数据分级授权：** 用户明确选择 L1-L4 采集级别
2. **敏感词过滤：** 自动过滤密码、账号等敏感信息
3. **数据本地优先：** 默认全部存储在本地
4. **一键清除：** 随时可删除所有历史数据
5. **AI 分析透明：** 明确告知哪些数据会发送到云端
6. **脱敏处理：** 上传云端前自动脱敏（如隐藏具体文件名）

**时间线导出功能：**

| 格式 | 用途 |
|------|------|
| JSON | 数据备份、第三方工具分析 |
| CSV | Excel 分析、自定义统计 |
| Markdown | 日报/周报生成 |
| PDF | 打印存档 |

**典型使用场景：**

> "我今天在电脑前坐了一整天，但好像什么都没完成？"
> → 打开时间线，发现微信沟通占了 2 小时，B站刷了 1.5 小时
> → 调整策略：集中处理微信，番茄钟严格限制娱乐

> "我想复盘这周的工作效率"
> → 导出周报，AI 分析显示：周二效率最高，周五下午容易分心
> → 调整：周二安排重要任务，周五下午安排轻松工作

#### 2.2.9 统计与报告（原有功能，整合行为日志）

**功能描述：** 提供详细的专注数据分析，帮助用户了解自己的专注模式。

**数据维度：**

1. **今日概览：**
   - 专注总时长
   - 分心次数
   - 公仔当前状态
   - 今日目标完成度
   - 番茄钟完成数

   - 专注时长趋势图
   - 分心应用 Top 5

   - 应用使用占比饼图（新增 v1.1）

   - 等级进度

4. **行为轨迹报告（新增 v1.1）：**

   - 按应用/任务/时段的多维统计
   - AI 洞察与改进建议（需授权）
   - 导出为 JSON/CSV/Markdown/PDF

## 三、非功能需求

### 3.1 隐私与安全

**核心原则：用户数据绝对本地化，AI 分析需明确授权**

| 数据类型 | 存储位置 | 是否上传 | 备注 |
|---------|---------|---------|------|
| 摄像头画面 | 仅内存，不存储 | ❌ | 本地 AI 分析后立即丢弃 |
| 注意力检测结果 | 本地数据库 | ❌ | 仅存储状态（专注/分心/离开）|
| 应用使用记录 | 本地数据库 | ⚠️ 可选 | L1-L4 分级授权 |
| 窗口标题/URL | 本地数据库 | ⚠️ 可选 | 需 L2+ 授权，敏感词自动过滤 |
| 行为轨迹日志 | 本地数据库 | ⚠️ 可选 | AI 分析时可能上传（需明确授权）|
| 用户设置 | 本地配置文件 | ❌ | - |
| 公仔数据 | 本地数据库 | ❌ | - |

**隐私声明要点：**
- 摄像头画面仅用于本地 AI 分析，不存储、不上传
- 用户可随时关闭摄像头监控
- 行为日志采集分级授权，默认仅采集 L1（应用名称+时长）
- AI 分析需用户明确授权，可选择本地 LLM 或云端 LLM
- 提供数据导出/删除功能
- 敏感信息（密码、账号等）自动过滤

### 3.2 性能要求

| 指标 | 要求 |
|------|------|
| 摄像头处理延迟 | < 500ms |
| 应用切换检测延迟 | < 100ms |
| CPU 占用（待机时） | < 5% |
| 内存占用 | < 200MB |
| 应用启动时间 | < 3 秒 |

### 3.3 兼容性要求

**首发平台：** macOS 11+（可后续扩展至 Windows）

**系统权限要求：**
- 摄像头权限
- 屏幕录制权限（用于窗口标题检测）
- 辅助功能权限（用于窗口切换检测）
- 通知权限
